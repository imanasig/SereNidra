## Project Overview

### Description

Sleep Meditation Generator is a web application that generates personalized meditation scripts based on user preferences (meditation type, duration, focus areas) using AI. Users can select their meditation type (stress relief, sleep, focus), receive a customized script generated by an AI agent, and generate audio narration with voice style preferences using Gemini TTS API. The application tracks meditation history to help users maintain a consistent practice.

### Target Users

- Individuals seeking guided meditation experiences
- People looking for personalized meditation content
- Users who want to track their meditation practice
- Anyone needing stress relief, better sleep, or improved focus

### Core Value Proposition

Provides personalized, AI-generated meditation scripts tailored to individual needs and preferences, eliminating the need to search through generic meditation content. Users get customized scripts on-demand and can track their meditation journey over time.

## API Endpoints Reference

### Base URL
```
http://localhost:8000
```

### Authentication
All protected endpoints require Firebase authentication token in the request headers.

### Endpoints

#### 1. Generate Meditation Script
```
POST /api/meditations/generate
```
**Protected:** Yes  
**LLM Integration:** Yes (LangChain + Gemini LLM)

**Request Body:**
```json
{
  "type": "sleep" | "stress" | "focus",
  "duration": 10 | 15 | 20 | 30,
  "preferences": {
    "focus": "string",
    "style": "string",
    "tone_accent": "Calm, soothing whisper" | "Clear, steady voice" | "Calm, relaxed tone"
  }
}
```

**Response:**
```json
{
  "id": "string",
  "script": "string",
  "type": "string",
  "duration": 10
}
```

---

#### 2. Get All Meditation Sessions
```
GET /api/meditations
```
**Protected:** Yes  
**LLM Integration:** No

**Response:**
```json
[
  {
    "id": "string",
    "type": "string",
    "duration": 10,
    "createdAt": "2024-01-01T00:00:00Z",
    "script": "string"
  }
]
```

---

#### 3. Get Single Meditation Session
```
GET /api/meditations/:id
```
**Protected:** Yes  
**LLM Integration:** No

**Response:**
```json
{
  "id": "string",
  "type": "string",
  "duration": 10,
  "script": "string",
  "createdAt": "2024-01-01T00:00:00Z",
  "tone_accent": "string",
  "audio_url": "string",
  "voice_used": "string",
  "audio_generated_at": "2024-01-01T00:00:00Z"
}
```

---

#### 4. Delete Meditation Session
```
DELETE /api/meditations/:id
```
**Protected:** Yes  
**LLM Integration:** No

**Response:**
```json
{
  "success": true,
  "message": "Meditation session deleted successfully"
}
```

---

#### 5. Search Meditation Sessions
```
GET /api/meditations/search?query=text&type=stress
```
**Protected:** Yes  
**LLM Integration:** No

**Query Parameters:**
- `query` (optional): Search text
- `type` (optional): Filter by meditation type

**Response:**
```json
[
  {
    "id": "string",
    "type": "string",
    "matches": "string"
  }
]
```

---

#### 6. Update Meditation Session
```
PUT /api/meditations/:id
```
**Protected:** Yes  
**LLM Integration:** No

**Request Body:**
```json
{
  "notes": "string",
  "rating": 5,
  "tone_accent": "string"
}
```

**Response:**
```json
{
  "success": true,
  "updated": {
    "id": "string",
    "notes": "string",
    "rating": 5
  }
}
```

---

#### 7. Generate Audio from Script
```
POST /api/meditations/:id/generate-audio
```
**Protected:** Yes  
**LLM Integration:** Yes (Gemini TTS)

**Request Body:** None (empty body)

**Note:** The backend automatically fetches the meditation record by ID, extracts the script text and stored `tone_accent` from the database, and uses them to generate audio with Gemini TTS.

**Response:**
```json
{
  "audio_url": "string",
  "voice_used": "string",
  "audio_generated_at": "2024-01-01T00:00:00Z"
}
```

## Database Schema

### Tables

#### `meditations`
Stores meditation sessions and generated scripts.

**Essential Fields:**
- `id` (PRIMARY KEY) - Unique identifier
- `user_id` (TEXT) - Firebase user ID
- `type` (TEXT) - Meditation type (sleep, stress, focus)
- `duration` (INTEGER) - Duration in minutes
- `script` (TEXT) - Generated meditation script
- `created_at` (TIMESTAMP) - Creation timestamp

**Additional Fields:**
- `tone_accent` (TEXT, nullable) - User-provided tone/accent preferences (voice style)
- `audio_url` (TEXT, nullable) - URL/path to generated audio file
- `voice_used` (TEXT, nullable) - Which Gemini voice was used
- `audio_generated_at` (TIMESTAMP, nullable) - When audio was generated
- `notes` (TEXT, nullable) - User notes
- `rating` (INTEGER, nullable) - User rating (1-5)

**Note:** Students design the exact field names, data types, and additional fields as needed.

## Getting Started

### Prerequisites

- Python 3.12+
- UV Package Manager
- Node.js 18+
- Google API Key ([Get one here](https://makersuite.google.com/app/apikey))
- Firebase project ([Create one here](https://console.firebase.google.com/))

### Installation

1. **Clone the repository**
   ```bash
   git clone <repository-url>
   cd sleep-meditation-generator
   ```

2. **Backend Setup**
   ```bash
   cd Backend
   uv venv
   source .venv/bin/activate  # On Windows: .venv\Scripts\activate
   uv add -r requirements.txt
   ```

3. **Create `.env` file**
   ```bash
   GOOGLE_API_KEY=your_api_key_here
   ```

4. **Frontend Setup**
   ```bash
   cd Frontend
   npm install
   ```

5. **Configure Firebase**
   - Create Firebase project
   - Enable Email/Password authentication
   - Add Firebase config to Frontend

6. **Run Development Servers**
   ```bash
   # Terminal 1 - Backend
   cd Backend
   uvicorn main:app --reload

   # Terminal 2 - Frontend
   cd Frontend
   npm run dev
   ```

### Access Points

- **Frontend:** http://localhost:5173
- **Backend API:** http://localhost:8000
- **API Documentation:** http://localhost:8000/docs

## Key Features Implementation

### Meditation Script Generation

1. User selects meditation type, duration, and preferences
2. Frontend sends request to `/api/meditations/generate`
3. Backend uses LangChain to create prompt
4. Gemini LLM generates personalized script
5. Script is returned and displayed to user
6. Session is automatically saved to database

### Audio Generation Flow

1. User clicks "Generate Audio" button (no voice style input needed)
2. Frontend sends POST request to `/api/meditations/:id/generate-audio` (empty body)
3. Backend fetches meditation record by ID from database
4. Backend extracts script text AND stored `tone_accent` from database
5. Backend uses Gemini TTS API with stored `tone_accent` as voice style prompt
6. Audio file (WAV) is generated and stored
7. Audio URL is saved to database
8. Frontend displays audio player with generated audio

### Session History

1. User navigates to History page
2. Frontend requests `/api/meditations`
3. Backend queries SQLite database
4. Sessions are displayed in chronological order
5. User can search, filter, view details, or delete sessions
